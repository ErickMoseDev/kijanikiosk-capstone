| **Task**                             | Create the Kubernetes manifest for the receipt-handler deployment in `kijani-staging`                                                                                                                                                                                                                                                                                                                                                               |
| **Prompt / context given**           | Asked to generate a `k8s-manifest.yaml` with a Deployment and Service for `receipt-handler` in the `kijani-staging` namespace, matching the port used by `serverless-offline` and including a readiness probe.                                                                                                                                                                                                                                      |
| **What AI produced**                 | A Deployment with `replicas: 1`, `containerPort: 3000`, a readiness probe on `GET /health:3000`, and a ClusterIP Service on port 3000. Resource requests/limits set at 50m CPU / 64Mi memory (requests) and 100m CPU / 128Mi memory (limits).                                                                                                                                                                                                       |
| **What it got right**                | Correct namespace; appropriate label selectors; readiness probe path matched the `httpApi` health function; Service port aligned with the container port.                                                                                                                                                                                                                                                                                           |
| **What it got wrong**                | Two issues: (1) `initialDelaySeconds: 5` was too short — `npx sls offline start` takes 15-30 seconds to load Serverless Framework, plugins, and the AWS SDK before the HTTP server is ready. (2) The memory limit of 128Mi was insufficient. Node.js + serverless-offline + serverless-s3-local + `@aws-sdk/client-s3` requires approximately 200-250MB at startup, causing the pod to be `OOMKilled` repeatedly before it could serve any traffic. |
| **What was changed before applying** | `initialDelaySeconds` raised from 5 to 30; `failureThreshold` added as 6; memory limit raised from 128Mi to 384Mi; memory request raised from 64Mi to 192Mi; `terminationGracePeriodSeconds: 10` added to reduce rollout wait time when replacing pods.                                                                                                                                                                                             |
